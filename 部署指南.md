# 打卡网站 - 部署指南

## 功能介绍
- ✅ iPhone 移动端适配
- ✅ 本地数据存储（私密）
- ✅ 跨设备同步（使用 GitHub Gist）
- ✅ 打卡记录、连续天数统计
- ✅ 7天打卡日历展示

## 免费部署方案推荐

### 方案一：GitHub Pages（推荐）
**优点**：完全免费、稳定、私密

**步骤**：
1. 在 GitHub 上创建一个新仓库（可以设为 Private）
2. 上传这三个文件：`index.html`、`style.css`、`script.js`
3. 进入仓库 Settings → Pages
4. Source 选择 `main` 分支，点击 Save
5. 几分钟后，你的网站就可以通过 `https://你的用户名.github.io/仓库名/` 访问了

### 方案二：Vercel
**优点**：部署简单、速度快

**步骤**：
1. 注册 Vercel 账号
2. 创建新仓库并上传文件到 GitHub
3. 在 Vercel 中导入该仓库
4. 自动部署完成

### 方案三：Netlify
**优点**：拖放上传即可

**步骤**：
1. 注册 Netlify 账号
2. 直接将文件夹拖放到 Netlify 网站
3. 自动获得一个可访问的链接

## 跨设备同步设置

### 1. 获取 GitHub Token
1. 访问 https://github.com/settings/tokens
2. 点击 "Generate new token" → "Generate new token (classic)"
3. 勾选 `gist` 权限
4. 生成并保存好这个 Token

### 2. 首次同步
1. 在手机上打开网站
2. 点击底部的「同步」按钮
3. 输入 GitHub Token，Gist ID 留空
4. 点击「上传」，会自动创建一个私有的 Gist
5. 复制生成的 Gist ID

### 3. 在其他设备同步
1. 在另一台设备上打开网站
2. 点击「同步」
3. 输入同样的 GitHub Token 和 Gist ID
4. 点击「下载」即可获取数据

## 注意事项
- 数据默认保存在浏览器本地，即使不使用同步功能也能正常使用
- GitHub Token 只保存在你的浏览器本地，不会发送到其他服务器
- Gist 设为私有，只有你能访问
